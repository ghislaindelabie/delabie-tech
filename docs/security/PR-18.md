## (2) Security-relevant decisions

1. **Avatar EXIF cleaned.** The source JPEG (a Lightroom-exported portrait) shipped with Canon camera serial numbers, lens serial, firmware version, and full Lightroom version string. These were stripped with `exiftool -all=` and only author-identity metadata (Title, Description, Artist, Creator, Copyright) was re-embedded. Minor data-leakage hardening; not a confirmed vulnerability but not intentional data-sharing either.

2. **YouTube embed uses the privacy-enhanced endpoint.** `_includes/publication-item.html:28` constructs the iframe `src` against `https://www.youtube-nocookie.com/embed/{id}`. The previous Phase guidance (no third-party cookies before consent) is preserved. Added `referrerpolicy="strict-origin-when-cross-origin"` and `loading="lazy"`.

3. **Iframe `title` attribute now escaped.** `_includes/publication-item.html:29` uses `| escape` on `include.pub.title` when it's interpolated into the `title=` attribute, so a future title containing `"` won't break out of the attribute quote. Defense-in-depth since titles are author-controlled.

4. **Bilingual co-author label.** `_includes/publication-item.html` now branches on `include.pub.lang` for the `with`/`avec` string. No security impact — closes a cross-cultural rendering bug flagged by code review.

5. **External resource links.** `_includes/teaching-item.html` emits external links (`external_url`, `video_url`) with `target="_blank" rel="noopener noreferrer"`; internal links (`pdf`, `slides`) use `relative_url` without `target` (same-origin, no tabnabbing vector).

6. **No new workflow permissions.** Phase 4 touches no `.github/workflows/*` files. CI gate unchanged.

7. **No new dependencies.** Gemfile and package.json unchanged.

8. **Sort key invariant.** The teaching collection now sorts on the scalar `year_end` frontmatter field rather than on the `years` array. A new structural test (`teaching_collection_spec.rb`) asserts `year_end` is a 4-digit integer matching `years.last` for every entry, so a future entry can't silently break the build or produce surprising ordering.

### Why zero

- **Iframe `src` is build-time constant.** The only dynamic segment is `youtube_id`, which a structural test confirms is a bare ID (no slashes, no schemes), not a URL.
- **External links are tabnabbing-safe.** All new `target="_blank"` anchors carry `rel="noopener noreferrer"`.
- **Avatar metadata is clean.** Camera-identifying EXIF was stripped; only author-identity fields remain.
- **No untrusted inputs.** The site has no forms, no auth, no server.
- **No new workflows / secrets / dependencies.** Attack surface unchanged from Phase 3.

## (5) Code-review findings also addressed in this PR

Not security-scoped but surfaced by the local `code-reviewer` subagent against the same diff; fixes landed on the same branch:

- **[H1] teaching-item.html — empty `years` span.** A Liquid filter-chain bug (`years | first | append: '–' | append: (years | last)`) produced empty strings because Liquid does not support parenthesized sub-expressions. Fixed with explicit `capture`. A Playwright assertion was added so `[data-test="teaching-years"]` must match `/\d{4}/` to prevent regression.
- **[H2] `collections.teaching.sort_by: years`.** Sorting on an array is fragile; switched to scalar `year_end` with a structural test enforcing `year_end == years.last`.
- **[M1] Hardcoded English/French labels.** Publication "avec"/"with" and teaching resource labels (`Course page`/`Slides`/`Video`) are now branched on `lang`.
- **[M2] Avatar EXIF + aspect ratio.** Portrait was 512×341 (would render squashed in Chirpy's circular sidebar); re-cropped from the hi-res source to 512×512 and camera serials stripped.
- **[M4] CV spec heading asymmetry.** `REQUIRED_FR_HEADINGS` now mirrors the EN set.
- **[M5] Dead CSS selector** in `phase-4.scss` removed.
- **[M6] Playwright mirror tests.** `FR count === EN count` could both be zero; now requires both `> 0`.
