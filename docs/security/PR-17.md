## (2) Security-relevant decisions

1. **External repo URLs now open in a new tab with `rel="noopener noreferrer"`.** `_includes/repo-card.html` was updated per [REVIEW-6]. Previously `rel="noopener"` was inert without `target="_blank"`; now both are set, preventing tabnabbing from the target origin (even though all targets here are github.com).

2. **Activity-item external-link handling.** `_includes/activity-item.html` detects absolute URLs (`contains '://'`) and applies `target="_blank" rel="noopener noreferrer"` only to external links. Internal links keep normal `<a>` behaviour. Prevents both `noopener`-without-`_blank` and same-window navigation on external references.

3. **No new workflow permissions.** Phase 3 touches no `.github/workflows/*` files. CI gate is still only `Build + structural + links + Playwright`.

4. **No new dependencies.** Gemfile and package.json unchanged.

5. **Data-source trust.** All data rendered by the new templates comes from three places: repo-tracked `_data/repositories.yml`, repo-tracked `_activity/*.md` frontmatter, and repo-tracked `_data/i18n.yml`. Changes to these go through the same PR + review process as code changes. No external sync.

6. **Markdown rendering.** `_includes/activity-item.html` renders entry content via `markdownify`. Kramdown's default configuration escapes raw HTML in Markdown unless `{::options parse_block_html="true" /}` is set — which we do not. An activity entry with a `<script>` tag in its body renders as escaped text, not executable markup.

### Why zero

- **No new trust-boundary crossings.** Templates only render repo-controlled data; no URL parameters, no form handlers, no postMessage listeners.
- **Liquid auto-escapes.** Every interpolation (`{{ include.entry.title }}`, `{{ repo.description }}`, `{{ cs.summary }}`) is auto-escaped by Jekyll's default Liquid. No `| raw` or `| safe` filters introduced.
- **Markdown is sandboxed.** `markdownify` in `activity-item.html` goes through Kramdown with raw-HTML disabled by default. A malicious activity entry can't inject script via markdown.
- **External-link tabnabbing closed.** Both new partials that emit external `<a target="_blank">` set `rel="noopener noreferrer"`.
- **No untrusted inputs.** The site has no forms, no auth, no server. All content is author-curated.
- **No new workflows / secrets / dependencies.** Attack surface unchanged.

The Phase 3 work is a pure content + presentation layer addition. No security posture change.
